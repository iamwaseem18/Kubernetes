# 🧪 Kubernetes Demo: Pods with YAML

This guide shows how to create a **Kubernetes Pod using a YAML file** instead of using `kubectl run`.

---

## 🔹 Why use YAML instead of `kubectl run`?

* Better **control over configuration**
* Easy to **edit and reuse**
* Supports **version control (GitOps)**

---

## 🔹 Step 1: Create YAML File

```bash
vim pod.yaml
```

---

## 🔹 Pod YAML Definition

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: nginx
  labels:
    app: nginx
    tier: frontend
spec:
  containers:
    - name: nginx
      image: nginx
```

---

## 🔹 Key Fields Explained

| Field      | Purpose                 |
| ---------- | ----------------------- |
| apiVersion | Kubernetes API version  |
| kind       | Resource type (Pod)     |
| metadata   | Name + labels           |
| spec       | Container configuration |

---

## 🔹 Important YAML Rules

* Use **spaces, not tabs**
* Maintain **consistent indentation**
* YAML is **case-sensitive** (`Pod` ≠ `pod`)

---

## 🔹 Step 2: Verify YAML

```bash
cat pod.yaml
```

👉 Check formatting before applying

---

## 🔹 Step 3: Create Pod

```bash
kubectl apply -f pod.yaml
```

👉 Output:

```bash
pod/nginx created
```

---

## 🔹 Step 4: Check Pod Status

### List Pods

```bash
kubectl get pods
```

---

### Detailed Info

```bash
kubectl describe pod nginx
```

---

## 🔹 Pod Lifecycle

Initial state:

```text
ContainerCreating
```

Final state:

```text
Running
```

---

## 🔹 Debugging (IMPORTANT)

If something fails:

```bash
kubectl describe pod nginx
kubectl logs nginx
```

👉 Check:

* Events
* Image issues
* Container errors

---

## 🔹 create vs apply (INTERVIEW GOLD)

| Command | Usage                         |
| ------- | ----------------------------- |
| create  | Only creates new resources    |
| apply   | Creates + updates (preferred) |

👉 Real-world:
Use **`apply`** for declarative workflows

---

## 🔹 Why this matters

* YAML is core to **Kubernetes**
* Used in **real deployments**
* Required for **DevOps / Platform roles**

---

## 📌 Summary

* Define Pods using YAML
* Use `kubectl apply` to deploy
* Use `get`, `describe`, `logs` to verify
* YAML ensures consistency and scalability
